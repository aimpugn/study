# go run

## `go run file.go` vs `go run .`

두 명령어 사이에 차이가 있는 이유는, Go 언어의 *빌드 시스템*과 *패키지 관리 방식* 때문이다.
이 두 명령은 Go 코드를 빌드하고 실행하는 방식에서 중요한 차이를 보인다

`go run file.go`는 `file.go` 파일만을 컴파일하고 실행하기 때문에, 다른 파일에 정의된 코드를 참조할 수 없다.
반면에 `go run .`는 현재 디렉토리 내의 모든 Go 파일을 컴파일하고 실행하기 때문에, `file.go` 파일이 다른 파일들과 상호작용하는 경우에 적합하다.

### **`go run file.go` 명령**

- 이 명령은 지정된 단일 파일(`file.go`)만을 컴파일하고 실행한다
- `file.go` 파일에 정의되어 있지 않고, **다른 파일에서 정의된 함수, 변수, 타입 등은 이 명령으로는 참조할 수 없다**.
- 만약 `main.go` 파일이 다른 파일에 정의된 패키지나 식별자를 사용하고 있다면, 그 파일은 컴파일 대상에 포함되지 않아 `undefined` 오류가 발생합니다.

### **`go run .` 명령**

- 이 명령은 **현재 디렉토리(`.`)를 기준으로 해당 디렉토리 내의 모든 Go 파일을 컴파일 대상으로 포함**한다.
- 따라서, `main.go` 파일이 다른 파일에 정의된 함수나 변수를 사용하고 있다면, 이들도 함께 컴파일되어 실행된다.
- 이 방식은 프로젝트 내의 여러 Go 파일들이 서로 상호작용하는 경우에 적합하다.
