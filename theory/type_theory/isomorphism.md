# isomorphism

## isomorphism(동형사상)?

동형사상이란 *서로 다른 형태를 가지지만, 동일한 정보를 표현할 수 있는 경우*를 말합니다.

"isomorphic"이라는 용어는 주로 수학적 구조 사이의 관계를 설명할 때 사용됩니다.
타입 이론이나 프로그래밍에서는 이와 비슷하면서도 약간 다른 의미로 "type equivalence"라는 용어를 사용할 수 있습니다.

수학에서 **동형사상(isomorphism)**은 수학적 구조에서 두 개체가 서로 같은 구조를 가질 때 이를 변환할 수 있음을 의미합니다.
즉, 한 구조에서 다른 구조로의 구조를 보존하는 일대일 대응을 말합니다.
이는 두 구조가 본질적으로 동일한 정보와 연산을 가지고 있음을 의미합니다.
개념은 프로그래밍에서도 자주 사용되며, 특히 타입 이론과 함수형 프로그래밍에서 중요합니다.

예를 들어, 두 그룹이 동형이면 하나의 그룹에서의 연산 결과를 다른 그룹에 그대로 적용할 수 있습니다.

### vs Type Equivalence

프로그래밍에서는 **타입 동등성(type equivalence)**이라는 개념이 종종 사용됩니다.
이는 두 타입이 같은 종류의 값과 연산을 수행할 수 있는지를 설명할 때 사용됩니다.
타입 동등성은 두 가지 주요 형태로 나뉩니다.

- 이름 동등성(nominal type equivalence)

    타입이 이름으로만 결정됩니다.
    즉, 동일한 이름을 가진 타입만이 동등합니다.

- 구조적 동등성(structural type equivalence)

    타입의 구조(예: 필드의 타입과 순서)가 동등성을 결정합니다.
    이름은 중요하지 않습니다.
