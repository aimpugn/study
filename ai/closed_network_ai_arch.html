플랫폼 제어 영역(GitLab)과 앱 데이터 영역(MySQL)의 물리적 격리를 포함한 이중화 MVP

배포 파이프라인 시퀀스 (Main Sequence)

01. Claude가 연구/개발망에서 개발 보조 (내부 접근 X)
09. DMZ Artifact Receiver (HA)가 L4를 통해 Upload 수신
02. 개발자가 코드 작성 후 Commit & Push
10. 무결성 확인 후 Shared Drop Zone(IDC 자원)에 영구 보관
03. GitLab (HA)에서 MR 생성, 코드 리뷰 후 Merge
11. Jenkins Controller (Active)가 배포 가능한 후보 인식
04. GitLab Pipeline Trigger 발생
12. 승인자가 Jenkins에서 배포 타이밍 선택 (수동 배포)
05. Runner가 별도의 R&D MySQL을 활용하여 통합 테스트 수행
13. Jenkins Agent가 서비스 계정(sudo su -s) 권한으로 타겟 스크립트 실행
06. Runner가 Nexus (HA)를 통해 의존성 다운로드
14. 롤링 배포 (L4 Drain ➔ Script Execution ➔ Health Check ➔ Join)
07. 테스트 통과 시 배포용 Artifact Bundle 및 Manifest 생성
15. Jenkins 화면에서 최소화된 배포 결과 및 상태 모니터링
08. Runner가 Bundle을 DMZ로 POST 전송
