## fuzz?

이 단어의 일반적인 의미는 "퍼지다, 모호하게 하다"와 같은 뜻을 지니지만, 소프트웨어 개발에서의 사용법은 좀 더 구체적이고 기술적인 의미를 갖는다.

[Fuzzing](https://en.wikipedia.org/wiki/Fuzzing)은 소프트웨어 테스팅의 한 방법으로, 예측 불가능한 상황을 다루거나 프로그램에 무작위 데이터(퍼지 데이터)를 입력하여 실행시켜 오류, 취약점, 버그 등을 찾아내는 과정이다. 이 방식은 일반적인 테스트 케이스로는 발견하기 어려운 예외적인 상황이나 입력에 대한 프로그램의 반응을 탐색한다.

## Fuzz의 의미 및 유래

### 의미

소프트웨어 엔지니어링에서 "fuzz testing" 또는 "fuzzing"은 프로그램에 대해 예상치 못한, 무작위적 또는 반 자동화된 데이터를 입력하여 프로그램이 오류, 버그, 보안 취약점 등을 발견하도록 하는 테스트 방법이다.
이 방식은 프로그램이 다루기 어려운 예외적 상황을 처리하는 방법을 평가하는 데 유용하다.

### 유래

Fuzzing의 개념은 1980년대 초반에 벨 연구소의 Barton Miller 교수와 그의 학생들이 처음 개발했다.
그들은 유닉스 운영 체제의 명령줄 도구들에 임의의 입력을 제공하여 프로그램이 어떻게 반응하는지 관찰했다.
소프트웨어가 예기치 않은 입력이나 극단적인 상황에 어떻게 반응하는지를 이해하고, 그 결과를 통해 소프트웨어의 안정성과 신뢰성을 개선하고자 하는 것이 목적이었다.
이 초기 실험에서 많은 표준 유틸리티들이 비정상적으로 종료되거나 실패하는 것을 발견했고, 이로부터 fuzzing이라는 개념이 탄생했다.

## Fuzzing의 핵심 요소

- 자동화된 테스팅: Fuzzing은 주로 자동화된 도구를 사용하여 수행된다. 이 도구들은 무작위 데이터를 생성하고, 이를 프로그램에 입력한 후 결과를 관찰한다.
- 무작위 데이터 생성: Fuzzing의 중요한 특징은 무작위성이다. 테스팅 도구는 일반적인 사용 사례와는 다른, 예측 불가능한 데이터를 생성하여 프로그램을 테스트한다.
- 에러 및 취약점 탐지: Fuzzing의 주된 목적은 프로그램이 무작위 데이터를 처리하면서 발생할 수 있는 에러나 취약점을 찾아내는 것으로, 보안 분야에서 특히 중요하다.
- 커버리지 기반 접근법: 많은 현대의 fuzzing 도구들은 코드 커버리지 정보를 사용하여 보다 효율적으로 데이터를 생성한다. 이를 통해 코드의 다양한 부분이 테스트될 수 있도록 한다.

## 개발에서의 Fuzz 사용 케이스

### 보안 취약점 탐지

Fuzzing은 보안 취약점을 찾는데 자주 사용된다
무작위 데이터를 입력함으로써 개발자는 예상치 못한 상황에서의 시스템 반응을 확인할 수 있으며, 이는 보안 허점을 발견하는 데 중요하다.

### 소프트웨어 안정성 검증

Fuzz 테스트는 소프트웨어가 예외적인 상황이나 잘못된 입력을 어떻게 처리하는지 확인하는 데 사용된다.
이를 통해 소프트웨어의 안정성을 높일 수 있습니다.

### 리그레션 테스팅

새로운 코드 변경으로 인해 예전에 발견되었던 버그가 재발하는 것을 방지하기 위해 사용된다.

### 성능 문제 탐지

예기치 않은 입력으로 인한 성능 저하나 자원 과소비를 탐지하는 데 사용된다.

Go Fuzzing은 코드 커버리지를 기반으로 하여 효율적으로 테스트 케이스를 생성합니다.

## Fuzz Sort

이는 데이터 정렬 알고리즘을 테스트하기 위해 사용되는 용어.
무작위로 생성된 데이터 집합을 사용하여 정렬 알고리즘이 모든 가능한 입력에 대해 올바르게 작동하는지 확인한다.

## FZF (Fuzzy Finder)

이는 명령줄에서 파일, 명령어 기록, 프로세스 등을 빠르게 검색하고 선택하는 도구.
여기서 'fuzzy'는 정확한 문자 일치가 아닌 유사한 결과를 반환하는 검색 방식을 의미한다.
