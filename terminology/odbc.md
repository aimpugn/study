## ODBC(Open Database Connectivity, 오픈 데이터베이스 연결성)?

ODBC(Open Database Connectivity, 오픈 데이터베이스 연결성)는 다양한 데이터베이스 시스템에 대한 표준 API(Application Programming Interface, 응용 프로그래밍 인터페이스)입니다.
이 표준을 사용하면, 하나의 인터페이스를 통해 서로 다른 데이터베이스 시스템에 접근할 수 있습니다.
즉, 개발자는 ODBC를 사용하여 다양한 데이터베이스 관리 시스템(DBMS)에 독립적인 프로그램을 작성할 수 있습니다.

ODBC는 마이크로소프트에 의해 개발되었으며, SQL 데이터베이스와의 통신을 위한 표준 방법을 제공합니다. ODBC를 사용하면, 애플리케이션은 데이터베이스의 종류나 사용하는 운영 체제에 관계없이 데이터베이스에 접근할 수 있습니다. 이는 데이터베이스 연결을 위한 유연성과 호환성을 크게 향상시킵니다.

## ODBC 구성 요소

- **응용 프로그램**: 사용자의 요청을 받아 ODBC 함수 호출을 통해 데이터베이스 작업을 수행합니다.
- **드라이버 매니저**: ODBC 함수 호출을 적절한 데이터베이스 드라이버로 전달합니다.
- **데이터베이스 드라이버**: 드라이버 매니저로부터 받은 호출을 해당 데이터베이스의 쿼리 언어로 변환합니다.
- **데이터 소스**: 실제 데이터가 저장된 데이터베이스입니다.
