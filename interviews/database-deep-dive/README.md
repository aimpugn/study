# Database interview deep dive

이 디렉터리는 데이터베이스 주제를 면접 준비용 정식 학습 자산으로 다시 구성하는 canonical 위치다.

`study/database` 아래의 기존 자료는 버릴 원고가 아니라 source다. 다만 source 문장 하나가 맞는다고 해서, 그 문장들을 모아 만든 큰 설명이 자동으로 맞아지지는 않는다. 이 디렉터리의 문서는 source 문장을 claim 단위로 검토하고, 그 claim들이 한 문서 안에서 어떤 순서와 경계로 엮일 때 올바른 mental model을 만드는지 다시 검토한 뒤에만 정식 본문으로 승격한다.

`study/database/deep-dive`는 이전 실행에서 만들어진 장문 초안이자 coverage 참고 자료다. 그 위치의 문서를 그대로 옮기거나 문장만 다듬어 정식 자산으로 보지 않는다. 정식 문서는 이 디렉터리에서 새로 작성하며, 기존 초안에서 가져오는 내용도 `audit/claim-audit.tsv`와 `audit/composition-audit.tsv`를 통과해야 한다.

## 작성 원칙

각 문서는 하나의 면접 질문 또는 하나의 판단 축을 중심으로 쓴다. 앞부분에는 2분에서 5분 정도로 말할 수 있는 개요를 두고, 본문에서는 작은 작동 단위에서 시작해 실제 메커니즘, DBMS별 경계, 직접 재생해 보는 방법까지 이어 간다. 꼬리 질문, 함정 질문, 더 깊게 볼 자료는 본문 뒤에 둔다.

기계 검증은 구조와 금지 패턴을 확인하는 보조 수단이다. 좋은 문서인지는 결국 claim의 근거, 전체 논리 흐름, 오해 방지, 직접 재생 가능성, 자연스러운 한국어 설명이 함께 통과해야 한다.

## 현재 상태

현재 상태는 `PARTIAL`이다. 첫 파일럿 문서로 `wal-redo-undo-crash-recovery-pitr.md`를 작성해, source 문장 검토와 composition 검토가 실제로 어떤 모양이어야 하는지 고정한다. 전체 DB 면접 심화 코퍼스는 아직 complete가 아니다.
