## 문제 해결 접근 방법

### 1. 문제 이해하기

문제의 요구사항, 입력/출력 예시, 제한 조건을 정확히 이해한다.

### 2. 문제 해결 전략 구상

문제를 해결하기 위한 기본 전략을 구상한다. 이때, 브루트 포스 방법부터 시작해서 최적화 방안을 찾아나갈 수도 있고, 알려진 알고리즘/자료구조가 문제에 적용 가능한지 생각해볼 수 있다.

### 3. 설계

어떤 변수가 필요한지, 어떤 자료 구조를 사용할 것인지, 어떤 함수/메서드가 필요한지 등을 생각한다.

### 4. 코딩

실제로 코드를 작성한다. 단계별로 작성하고, 각 단계에서 작성한 코드가 정확한지 확인한다.

### 5. 테스트

여러 입력 예시로 코드를 테스트한다. 가능하다면, 엣지 케이스도 고려한다.

### 6. 디버깅과 최적화

코드에 문제가 있다면 디버깅을 하고, 문제가 없다면 추가적인 최적화를 고려한다.

## 시간 초과 문제

- 대체로 알고리즘의 복잡도나 불필요한 계산을 줄이는 것에서 시작된다

### Memoization

- 이미 계산된 결과를 저장해두고 재활용하는 기법입니다. 이미 dp 맵을 사용하고 있으나, 계산이 불필요한 경우에도 계속 재귀 호출이 이루어지고 있는 것 같습니다.

### Pruning

불필요한 계산을 줄이기 위해 어떤 조건을 체크하고 재귀 호출을 하지 않을 수 있습니다. 예를 들어, 목적지에 도달할 수 없을 것 같다는 것을 미리 알 수 있다면 그 경로는 더 이상 탐색하지 않아도 됩니다.

### Loop Unrolling

재귀 호출 대신 명시적인 스택을 사용하여 재귀를 제거할 수 있습니다. 하지만 이 경우에는 재귀의 깊이가 얼마나 될지, 그리고 그로 인한 성능 향상이 얼마나 될지 확인해야 합니다.

### Parallelization

이미 언급했듯이, Go의 goroutine을 사용하여 병렬 처리를 할 수 있습니다. 하지만 이 경우에는 동기화 문제와, 너무 많은 goroutine 생성으로 인한 오버헤드가 있을 수 있습니다.

### Early Exit

목표를 달성한 경우 더 이상의 탐색 없이 빠르게 종료할 수 있습니다.
