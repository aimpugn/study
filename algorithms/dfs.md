## DFS?

- Depth First Search, 말 그대로 **깊이**를 우선 탐색하는 알고리즘
- 노드와 엣지로 구성된 그래프나 트리를 탐색하기 위한 알고리즘 중 하나로, DFS는 그래프의 모든 노드를 방문하고자 할 때 사용된다
- 시작 노드에서 출발하여 갈 수 있는 경로가 있는 노드까지 최대한 깊게 들어가며 탐색을 진행
    - 경로가 더 이상 없거나 목표 노드에 도달하면 리턴하여 다른 경로를 탐색
    - 이 과정을 모든 노드를 방문할 때까지 반복

## 특징

- DFS는 모든 가능한 경로를 탐색한다. 따라서 탐색 공간이 넓고, 가능한 모든 경로를 고려해야 하는 문제에 적합하다
- DFS는 사이클이 없는 그래프(예: 트리)에서 효율적이다 사이클이 있는 경우 추가적인 로직이 필요하다
- 메모리 효율적이다. DFS는 탐색하는 경로만 기억하면 되므로 BFS보다 일반적으로 메모리 사용이 적다.
- 해가 존재한다면 찾는다. DFS는 모든 레벨의 노드를 탐색하기 때문에, 해가 존재한다면 반드시 찾을 수 있다
    - 단, 문제의 조건이나 구현 방법에 따라 DFS는 해를 찾지 못할 수도 있다
    - DFS가 깊이를 우선으로 탐색하다 보니, 어떤 조건을 만족하지 못하는 경로로 빠질 가능성이 있기 때문

## 장점

- 구현이 상대적으로 간단하다
- 공간 복잡도가 BFS보다 일반적으로 좋다
- 사이클을 찾거나, 경로의 존재 유무, 연결성 등을 체크하기 좋다

## 공간 복잡도: `O(V)`

- DFS 알고리즘의 공간 복잡도는 그래프의 형태와 탐색의 순서에 따라 다를 수 있다
- 일반적으로 `O(V)`의 공간 복잡도(V = 그래프의 노드 수)  
- 재귀 함수 호출이나 명시적인 스택을 사용하므로 그래프의 깊이에 따라 공간이 필요하다

## 시간 복잡도: `O(V + E)`

- DFS의 시간 복잡도는 모든 노드와 엣지를 한 번씩 방문하므로, `O(V + E)`
    - V = 노드 수
    - E = 엣지(간선) 수

## 함수의 콜 스택

- DFS를 구현할 때 재귀적으로 함수를 호출하게 된다. 이렇게 하면 시스템의 콜 스택을 이용해 알고리즘의 스택을 구현하게 된다.
- 이 때문에 별도의 스택 자료구조를 사용하지 않아도 되지만, 그래프가 너무 깊어지면 스택 오버플로우가 발생할 위험이 있다

## 구현

- 기본적으로 DFS의 특성상, 재귀 호출을 통해 탐색을 진행하고 결과를 반환한다
    - 가령 "탐색을 시작한 상태부터 목표 상태까지 도달하는 최단 경로"를 반환해야 한다

## 예제

### 기본 예제

```go
package main

import "fmt"

// DFS
// - `graph`: 전체 그래프
// - `visited`: 이미 방문한 노드를 체크하기 위한 map
// - `node`: 시작 노드
func DFS(graph map[string][]string, visited map[string]bool, node string) {
    if visited[node] {
        return
    }
    fmt.Println(node)
    visited[node] = true
    for _, n := range graph[node] {
        DFS(graph, visited, n)
    }
}

func main() {
    graph := make(map[string][]string)
    graph["A"] = []string{"B", "C"}
    graph["B"] = []string{"D", "E"}
    graph["C"] = []string{"F"}
    graph["D"] = []string{}
    graph["E"] = []string{"F"}
    graph["F"] = []string{}

    visited := make(map[string]bool)
    DFS(graph, visited, "A")
}
```
