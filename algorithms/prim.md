# prim

## 프림 알고리즘 (Prim's Algorithm)

**프림 알고리즘(Prim's Algorithm)**은 그래프 이론에서 **최소 신장 트리(Minimum Spanning Tree, MST)**를 찾는 **그리디 알고리즘** 중 하나입니다. 최소 신장 트리는 주어진 가중치 그래프에서 모든 정점을 연결하는 트리 중 **최소 비용으로** 연결된 트리입니다. 프림 알고리즘은 이 최소 신장 트리를 구하는 효율적인 방법으로, **점증적인 방식**으로 트리를 확장해 나갑니다.

## 1. **최소 신장 트리(MST)란?**

최소 신장 트리는 다음과 같은 조건을 만족하는 트리입니다:
- **연결 그래프**: 모든 정점이 연결된 그래프.
- **트리 구조**: 사이클이 없는 연결 그래프.
- **최소 가중치**: 그래프에서 가능한 모든 신장 트리 중에서 간선들의 가중치 합이 가장 작은 트리.

### 예시

다음과 같은 그래프가 있을 때:

```
  (A)
   | 3
  (B)---2---(C)
   | 1      |
  (D)---4---(E)
```

- A, B, C, D, E는 **정점(노드)**이고, 간선에 붙은 숫자는 **가중치**입니다.
- 이 그래프의 최소 신장 트리는, 모든 노드를 연결하는 트리 중에서 **가중치의 합이 가장 작은 트리**입니다.

## 2. **프림 알고리즘의 개요**

프림 알고리즘은 **시작점**에서부터 출발해 점차 트리를 확장하면서 최소 신장 트리를 구하는 방식입니다. 알고리즘의 핵심은 **현재까지 선택된 정점 집합에서 가장 적은 비용으로 연결될 수 있는 간선을 매번 선택하는 방식**입니다.

- **그리디 알고리즘**: 프림 알고리즘은 매번 **가장 작은 가중치의 간선**을 선택하는 그리디 방식을 사용합니다.

## 3. **프림 알고리즘의 작동 방식**

프림 알고리즘의 작동 과정을 단계별로 설명하면 다음과 같습니다:

1. **시작 정점 선택**: 임의의 정점에서 시작합니다. (보통 아무 정점이나 선택할 수 있습니다.)

2. **가장 적은 가중치의 간선 선택**: 현재 선택된 정점들로부터 연결될 수 있는 간선 중에서 **가장 작은 가중치를 가진 간선**을 선택합니다.

3. **새로운 정점 추가**: 선택한 간선에 연결된 정점이 **아직 방문되지 않은 정점**이라면, 그 정점을 트리에 추가합니다.

4. **반복**: 트리의 모든 정점이 포함될 때까지 2번과 3번 과정을 반복합니다.

## 4. **프림 알고리즘의 구체적 과정**

프림 알고리즘을 구체적으로 설명해 보겠습니다.

### 예시 그래프

```
    2        3
  A---B----C
   \   |   /
   6\  8| /5
     \  |/
      D
```

- 각 간선의 가중치가 표시된 그래프입니다. 정점 A에서 시작해 최소 신장 트리를 구성해 보겠습니다.

### 과정

1. **초기 상태**: 정점 A에서 시작합니다. 현재 트리는 `{A}`이며, 선택할 수 있는 간선은 A-B(2), A-D(6)입니다.

2. **최소 간선 선택**: A에서 가장 작은 가중치를 가진 간선은 A-B(2)입니다. B를 트리에 추가하고, 현재 트리는 `{A, B}`가 됩니다. 추가할 수 있는 간선은 B-C(3), B-D(8), A-D(6)입니다.

3. **다음 최소 간선 선택**: 현재 선택할 수 있는 간선 중 가장 작은 것은 B-C(3)입니다. C를 트리에 추가하고, 현재 트리는 `{A, B, C}`입니다. 추가할 수 있는 간선은 C-D(5), B-D(8), A-D(6)입니다.

4. **마지막 간선 선택**: 남은 간선 중에서 가장 작은 것은 C-D(5)입니다. D를 트리에 추가하면 트리는 `{A, B, C, D}`가 되며 모든 정점이 연결됩니다.

최종 결과:
- 선택된 간선: A-B(2), B-C(3), C-D(5)
- 최소 신장 트리의 총 가중치: 2 + 3 + 5 = 10

## 5. **프림 알고리즘의 의사 코드**

프림 알고리즘을 의사 코드로 표현하면 다음과 같습니다:

```
Prim(graph, start):
    1. Initialize a set MST to store the minimum spanning tree
    2. Initialize a priority queue (min-heap) to store edges based on their weights
    3. Add all edges starting from the start node to the priority queue
    4. While there are edges in the priority queue:
        5. Extract the edge with the smallest weight
        6. If the edge connects to a new node (not in MST):
            7. Add the edge and the connected node to MST
            8. Add all edges from the new node to the priority queue
    9. Return the MST
```

```java
// G = (V, E): 그래프
// r: 시작 정점
Prim(G, r):
{
    S <-∅; T <- ∅; // S: 정점 집합, T: 간선 집합
    정점 r을 집합 S에 더한다;
    while(S != V) {
        S에서 V-S를 연결하는 간선들 중 최소 길이의 간선 (x, y)를 찾는다; // x∈S, y ∈ V-S

        T에 간선 (x, y)를 더한다;
        정점 y를 집합 S에 더한다;
    }
}
```

## 6. **프림 알고리즘의 시간 복잡도**

- **O(E log V)**: 프림 알고리즘은 **우선순위 큐(Priority Queue)**를 사용하여 간선을 선택하기 때문에, 시간 복잡도는 그래프의 간선 수(E)와 정점 수(V)에 따라 결정됩니다.
    - 각 간선을 우선순위 큐에 삽입하는 데 O(log V)의 시간이 걸리며, 이 작업을 모든 간선에 대해 수행하므로 전체 복잡도는 O(E log V)가 됩니다.

## 7. **프림 알고리즘의 장점**

- **그리디 방식**으로 매번 가장 작은 간선을 선택하기 때문에 구현이 직관적입니다.
- **연결된 그래프**에서 작동하며, 시작점을 자유롭게 선택할 수 있습니다.
- 프림 알고리즘은 **밀집 그래프**에서 효율적입니다. 왜냐하면 간선이 많은 그래프에서도 우선순위 큐를 사용하여 최소 가중치를 가진 간선을 빠르게 선택할 수 있기 때문입니다.

---

## 8. **프림 알고리즘과 크루스칼 알고리즘의 비교**

최소 신장 트리를 구하는 또 다른 유명한 알고리즘은 **크루스칼 알고리즘(Kruskal’s Algorithm)**입니다. 두 알고리즘의 차이를 살펴보면:

| 특성               | 프림 알고리즘 (Prim)                   | 크루스칼 알고리즘 (Kruskal)               |
|------------------|-------------------------------------|---------------------------------------|
| 작동 방식           | 시작 정점에서부터 점진적으로 트리를 확장해 나감 | 간선의 가중치를 오름차순으로 정렬한 뒤, 사이클이 없는 간선을 선택 |
| 효율성              | **밀집 그래프**에서 효율적               | **희소 그래프**에서 효율적                  |
| 구현 방식           | 정점에서 출발, 간선 선택                | 간선에서 출발, 간선 선택                     |
| 시간 복잡도         | O(E log V)                          | O(E log E)                             |

## 결론

프림 알고리즘은 **최소 신장 트리**를 찾기 위해 시작 정점에서부터 **가장 적은 가중치의 간선을 선택**하여 트리를 확장하는 방식입니다. 이 과정에서 그리디 알고리즘을 사용하여 항상 최적의 선택을 하도록 설계되었으며, 그래프 이론에서 매우 중요한 알고리즘입니다.

### 1. **의사 코드 설명**

프림 알고리즘은 주어진 그래프 G에서 시작 정점 r로부터 점진적으로 최소 신장 트리를 만들어가는 과정입니다.

#### 과정 설명

- **S**: 현재까지 선택된 정점들의 집합입니다.
- **T**: 최소 신장 트리에 포함된 간선들의 집합입니다.
- 알고리즘은 시작 정점 `r`에서 출발해, 아직 트리에 포함되지 않은 정점 중 **가장 작은 가중치의 간선**을 선택해 트리를 확장합니다.
- 트리에 포함된 정점들이 모든 정점이 될 때까지 위 과정을 반복합니다.

#### 의사 코드 설명 단계별 요약

1. **S, T 초기화**: 정점 집합 `S`와 간선 집합 `T`를 비어 있는 상태로 시작합니다.
2. **시작 정점 추가**: 시작 정점 `r`을 집합 `S`에 추가합니다.
3. **최소 가중치 간선 선택**: `S`에 있는 정점들에서 `V - S` (아직 선택되지 않은 정점들)로 연결되는 간선 중 가중치가 가장 작은 간선을 선택합니다.
4. **간선과 정점 추가**: 선택된 간선을 `T`에 추가하고, 해당 간선에 연결된 새로운 정점을 `S`에 추가합니다.
5. **반복**: 모든 정점이 `S`에 포함될 때까지 3번, 4번 과정을 반복합니다.

### 2. **프림 알고리즘의 Java 구현**

프림 알고리즘은 시작 정점에서 출발해, 최소 가중치 간선을 반복적으로 선택하여 트리를 확장하는 그리디 알고리즘입니다.
우선순위 큐(Priority Queue)를 활용하여 효율적으로 최소 가중치 간선을 선택하며, 이를 통해 최소 신장 트리를 구할 수 있습니다.

```java
import java.util.*;

/**
 * 간선의 목적지와 가중치를 나타내는 클래스로,
 * 가중치를 기준으로 우선순위 큐에서 정렬됩니다.
 */
class Edge implements Comparable<Edge> {
    int target;
    int weight;

    public Edge(int target, int weight) {
        this.target = target;
        this.weight = weight;
    }

    @Override
    public int compareTo(Edge other) {
        return this.weight - other.weight; // 간선의 가중치를 기준으로 오름차순 정렬
    }
}

public class Prim {
    /**
     * 시작 정점부터 최소 신장 트리를 구합니다.
     * `PriorityQueue`를 사용해 항상 최소 가중치 간선을 선택합니다.
     */
    static int prim(int[][] graph, int start) {
        int n = graph.length; // 그래프의 정점 수

        // 정점의 방문 여부를 기록합니다. 이미 선택된 정점을 표시하여 중복을 방지합니다.
        boolean[] visited = new boolean[n]; //
        PriorityQueue<Edge> pq = new PriorityQueue<>();

        pq.add(new Edge(start, 0));
        int totalWeight = 0; // 최소 신장 트리의 총 가중치

        while (!pq.isEmpty()) {
            Edge edge = pq.poll(); // 최소 가중치 간선 선택
            int v = edge.target;

            if (visited[v]) continue; // 이미 선택된 정점이라면 스킵
            visited[v] = true; // 정점 선택
            totalWeight += edge.weight; // 가중치 합산

            // 선택된 정점으로부터 연결된 모든 간선 추가
            for (int i = 0; i < n; i++) {
                if (!visited[i] && graph[v][i] != 0) {
                    pq.add(new Edge(i, graph[v][i]));
                }
            }
        }

        return totalWeight;
    }

    public static void main(String[] args) {
        int[][] graph = {
            {0, 2, 0, 6, 0},
            {2, 0, 3, 8, 5},
            {0, 3, 0, 0, 7},
            {6, 8, 0, 0, 9},
            {0, 5, 7, 9, 0}
        };

        int result = prim(graph, 0); // 시작 정점: 0
        System.out.println("최소 신장 트리의 가중치 합: " + result);
    }
}
```

그래프 구조는 다음과 같습니다.

```java
   (0)---2---(1)
    |      / | \
   6|    3  8|  5
    |  /     |   \
   (3)       (2)--7--(4)
      \       |
       9------/
```

1. **시작 정점 선택 (정점 0)**:
    - 시작 정점은 0입니다.
    - 선택 가능한 간선: (0, 1, 2), (0, 3, 6)
    - 최소 간선: (0, 1, 2)

    ```java
    S = {0, 1}
    T = {(0, 1, 2)}
    ```

2. **다음 최소 간선 선택**:
    - 선택 가능한 간선: (0, 3, 6), (1, 2, 3), (1, 4, 5), (1, 3, 8)
    - 최소 간선: (1, 2, 3)

    ```java
    S = {0, 1, 2}
    T = {(0, 1, 2), (1, 2, 3)}
    ```

3. **다음 최소 간선 선택**:
   - 선택 가능한 간선: (0, 3, 6), (1, 3, 8), (1, 4, 5), (2, 4, 7)
   - 최소 간선: (1, 4, 5)

    ```java
    S = {0, 1, 2, 4}
    T = {(0, 1, 2), (1, 2, 3), (1, 4, 5)}
    ```

4. **다음 최소 간선 선택**:
   - 선택 가능한 간선: (0, 3, 6), (1, 3, 8), (4, 3, 9)
   - 최소 간선: (0, 3, 6)

    ```java
    S = {0, 1, 2, 3, 4}
    T = {(0, 1, 2), (1, 2, 3), (1, 4, 5), (0, 3, 6)}
    ```

모든 정점이 선택되었으므로 알고리즘 종료합니다.
최종 최소 신장 트리는 다음과 같습니다.

```java
T = {(0, 1, 2), (1, 2, 3), (1, 4, 5), (0, 3, 6)}
최소 신장 트리의 가중치 합 = 2 + 3 + 5 + 6 = 16
```
